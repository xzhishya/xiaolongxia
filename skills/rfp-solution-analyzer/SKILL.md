# RFP Solution Analyzer

Read requirement-heavy documents and turn them into a response plan.

## Workflow

1. Identify the document type:
   - RFP / tender
   - requirement specification
   - SOW
   - draft proposal
2. Extract the essentials:
   - customer background and objective
   - scope of work
   - mandatory requirements
   - optional / bonus requirements
   - deliverables
   - timeline
   - qualification constraints
   - evaluation or scoring criteria
3. Separate requirements into three buckets:
   - clearly covered
   - partially covered / unclear
   - missing or risky
4. Highlight commercial, legal, delivery, and technical risks.
5. Convert the analysis into an execution-oriented response plan.

## Default Output

### Executive summary
- what the customer wants
- bid complexity / urgency
- overall fit assessment

### Requirement matrix
Use bullets or a compact table when appropriate:
- requirement
- priority: mandatory / important / optional
- response status: covered / partial / gap
- notes / evidence

### Scoring and win factors
- explicit scoring items
- likely hidden decision factors
- differentiators to emphasize

### Risks and gaps
- missing capability
- unclear requirement
- timeline / resource risk
- compliance or qualification risk

### Recommended proposal structure
Typical sections:
- customer understanding
- solution architecture / approach
- implementation plan
- project team
- delivery milestones
- security / compliance
- commercial response
- case studies / credentials

### Next actions
- clarifications to ask customer
- material to collect internally
- owners for each section
- deadline-critical tasks

## Rules

- Quote or paraphrase carefully; do not invent requirements.
- Mark ambiguous clauses as ambiguous.
- Call out every hard deadline and submission condition.
- Treat mandatory requirements as red-line items.
- If the user provides both an RFP and a draft proposal, compare them directly and show coverage gaps.

## Special Focus Areas

### For technical solutions
Check:
- architecture fit
- integration requirements
- security / deployment constraints
- data, AI, and model-related obligations

### For delivery planning
Check:
- timeline realism
- staffing assumptions
- acceptance criteria
- third-party dependencies
