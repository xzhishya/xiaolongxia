# Weekly Status Report

Convert scattered weekly inputs into a management-ready status report.

## Workflow

1. Identify the reporting scope:
   - sales
   - proposal / solution
   - project delivery
   - R&D / algorithm support
2. Extract the week window if available; otherwise state the assumed period.
3. Group information into:
   - completed this week
   - in progress
   - risks / issues
   - metrics or notable numbers
   - next week plan
   - support needed
4. Remove noise and merge duplicate updates from different sources.
5. Rewrite in an executive style: short, factual, outcome-first.

## Output Pattern

Use this default structure:

### Weekly summary
- 2-5 bullets on the most important outcomes

### Progress this week
- completed items
- important meetings / decisions
- customer or internal milestones reached

### Current status
- active items and percent / stage if known

### Risks and issues
- issue
- impact
- owner or proposed action

### Metrics
Include only if present, for example:
- opportunity count / pipeline amount
- proposal submissions
- milestone completion
- bug count / experiment count / model metrics

### Next week plan
- concrete planned actions

### Need support
- decisions, resources, approvals, or dependencies that require escalation

## Tone and Quality Bar

- Prefer crisp manager-readable bullets over long prose.
- Lead with outcomes, not activity.
- Flag slippage early.
- Distinguish fact from assumption.
- If data is incomplete, say what is missing.

## Variants

### Sales weekly
Add:
- new leads / active opportunities
- stage changes
- expected close risks
- key customer follow-ups

### Proposal / solution weekly
Add:
- active bids / proposals
- requirement changes
- review status
- submission deadlines

### Project weekly
Add:
- milestone status
- delivery confidence
- dependency and resource risk

### R&D / model weekly
Add:
- experiments run
- metric movement
- blockers in data, compute, or requirements
- recommended next experiments
